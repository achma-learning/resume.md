Règles strictes pour ce bloc :
  1. Si l'utilisateur a fourni un lien PDF → le reporter tel quel.
     Si c'est un upload local sans URL → mettre `upload local — [nom du fichier]`.
  2. Le chemin de la transcription est TOUJOURS reporté.
  3. Ne cite QUE les URLs réellement utilisées pour produire la fiche.
     Pas d'URL décorative / de remplissage.
  4. Vérifie la cohérence des URLs (pas de lien évidemment inventé).

[EXIGENCES DE FORME]
  • Langue : français médical, registre externat.
  • Format : Markdown pur, prêt à déposer dans le vault Obsidian.
  • Liens internes Obsidian [[…]] pour concepts/maladies/médicaments liés.
  • Frontmatter YAML complet (module, matiere, semestre, enseignant,
    source_pdf, tags) — renseigner `source_pdf: "[PDF_URL]"` si fourni.
  • Pas de bloc préambule ("Voici la fiche…"), pas de méta-commentaire :
    sors directement le contenu du fichier .md.
  • Première ligne = commentaire HTML avec nom de fichier suggéré :
    <!-- suggested filename: [topic-slug].md -->
  • Deuxième ligne = commentaire HTML avec destination :
    <!-- destination: [DESTINATION] -->
    (ex. s7/ped-chir/, s5/pharma/, s6/gastro/, s8/anapath/…)

[CONTRAINTES DE QUALITÉ]
  • Si une section ne peut pas être remplie faute de données dans les
    3 sources, écris explicitement :
      "⚠️ À compléter — donnée absente des sources fournies"
    NE JAMAIS inventer de chiffres, dosages, classifications.
  • Priorise la hiérarchisation examen : ce qui tombe > ce qui est rare.
  • Pour la pharmacologie : toujours DCI, classe, mécanisme, posologie
    adulte ET pédiatrique (si pertinent), CI, EI, surveillance.
  • Pour la chirurgie : indication, voie d'abord, temps opératoires clés,
    complications per- et post-opératoires, surveillance.
  • Pour l'anapath : macro + micro + IHC + critères diagnostiques +
    diagnostic différentiel histologique + grading/staging.
  • Pour la gastro / médecine interne : raisonnement clinique explicite
    (arguments pour / contre chaque hypothèse).
